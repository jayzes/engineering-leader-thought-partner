# /new-role

Start a conversation about transitioning into a new engineering leadership role.

## Trigger

User says `/new-role` or mentions starting a new EM, Director, VP, CTO, or staff role.

## Behavior

1. Understand the context:
   - What's the role?
   - Internal promotion or external hire?
   - What's the current state of the team/org?

2. Guide 90-day planning:
   - Phase 1: Listen (Days 1-30)
   - Phase 2: Diagnose (Days 30-60)
   - Phase 3: Act (Days 60-90)

3. Help identify:
   - Key relationships to build
   - Questions to ask
   - Documents to read
   - Early wins to target
   - Pitfalls to avoid

## Key Advice

- Learn before changing
- Build relationships first
- Ask "what should I not break?"
- Avoid big changes in month one
- Document what you're learning
